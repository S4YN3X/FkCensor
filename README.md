# FkCensor

Standalone-приложение для Windows/Linux/macOS, которое автоматически заменяет цензурированные треки в Яндекс Музыке на оригинальные версии. Работает без каких-либо дополнительных программ — один `.exe` файл.

> **Идея:** [Hazzz895](https://github.com/Hazzz895) — автор оригинального аддона [FckCensor](https://github.com/Hazzz895/FckCensor) для PulseSync  
> **Реализация:** [S4YN3X](https://github.com/S4YN3X) — переработка в самостоятельное Go-приложение



***

## Установка

1. Скачать последний релиз со страницы [Releases](../../releases)
2. Распаковать в любую папку
3. Запустить `FkCensor.exe`

Яндекс Музыка запустится автоматически. Иконка появится в системном трее.

> **Требования:** Яндекс Музыка (десктопная версия)

***

## Использование

После запуска иконка FkCensor появляется в трее. Правый клик открывает меню:

| Пункт | Описание |
|---|---|
| **Включить / Выключить** | Пауза и возобновление замен |
| **Открыть Яндекс Музыку** | Запустить ЯМ (если закрыта) |
| **Реинжектировать скрипт** | Повторно внедрить скрипт (после обновления ЯМ) |
| **Автозапуск с Windows** | Запускать FkCensor при входе в систему |
| **Выход** | Закрыть FkCensor |


***

## Как это работает

FkCensor запускает Яндекс Музыку в режиме отладки (`--remote-debugging-port=9222`) и через протокол CDP (Chrome DevTools Protocol) внедряет JavaScript прямо в страницу приложения. Этот скрипт перехватывает внутреннюю функцию получения аудиофайла и подменяет нужные треки на оригинальные версии, которые раздаёт встроенный HTTP-сервер.

```
FkCensor.exe
    │
    ├── Запускает ЯМ с --remote-debugging-port=9222
    ├── Подключается по WebSocket (CDP)
    │       └── Вставляет inject.js в страницу ЯМ
    │
    ├── HTTP-сервер
    │       ├── GET /replacements  ← JS получает список замен
    │       └── GET /track/{id}    ← ЯМ получает замененный трек
    │
    └── Watcher — следит что ЯМ запущена с правильным флагом
```

Список замен автоматически синхронизируется с [FckCensorData](https://github.com/Hazzz895/FckCensorData) каждые 10 минут.

***

## Известные ограничения

- При обновлении Яндекс Музыки скрипт может перестать работать из-за смены внутренних имён Webpack-модулей. В этом случае нажми **Реинжектировать скрипт** или дождись обновления
- На macOS и Linux автозапуск реализован в упрощённом виде
- Для работы необходима десктопная версия ЯМ (не веб-плеер)
